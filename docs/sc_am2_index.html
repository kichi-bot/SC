カテゴリ別傾向

詳細版:sc_am2_trend.md

暗号・ハッシュ・電子署名 A

毎回3問前後。同一問題の再出題が最も多く、暗記効率が最高の分野。

  • 典型問題:衝突発見困難性(6回)、CRYPTREC暗号リスト、暗号利用モード、MAC、サイドチャネル/タイミング攻撃、PQC(3回)
  • ひっかけ:ハッシュ/MAC/デジタル署名の役割混同。「誰の・何鍵か」の入れ替え
  • 対策:定番問題の正答文を暗記。鍵の主体を必ず意識
  • 午後とのつながり:中。証明書・TLS・危殆化対応の前提知識
PKI・証明書・TLS・HTTPS A

毎回1.5問前後。CRL/OCSP/RA/VA/CPSがローテーション出題される。

  • 典型問題:デジタル証明書の用途(6回)、OCSP(5回)、CRL、RA=本人確認、VA=失効応答、CPS=運用規程、HSTS
  • ひっかけ:CRLに載るのは「有効期限内に失効した」証明書。期限切れは載らない
  • 対策:CA/RA/VAの役割分担とCRL/OCSPの違いを比較で暗記
  • 午後とのつながり:高。サーバ証明書検証・TLS設定は午後頻出
認証・認可・SAML・OAuth A

毎回2問強で増加傾向(R6秋は6問)。SAML/OAuth/FIDOが熱い。

  • 典型問題:SAMLの流れ(IdP/SP/アサーション)、OAuth2.0のトークン発行者、FIDO/パスキー、802.1X+RADIUS、EAP-TLS、AAA
  • ひっかけ:認証(本人確認)と認可(権限付与)の混同
  • 対策:SAMLとOAuthの登場人物と流れを一度図に描く
  • 午後とのつながり:高。SSO・API認可・多要素認証は午後の定番
Web脆弱性 A

毎回1〜2問だが、午後との相乗効果が最大の分野。

  • 典型問題:CSRF対策で「効果がないもの」(3回)、OSコマンドインジェクションのログ読解、SQLi対策の組合せ、WAFのフォールスポジティブ
  • ひっかけ:XSS対策(エスケープ)をCSRF対策と誤認させる選択肢が定番
  • 対策:攻撃と対策を必ず対で覚える。仕組みまで理解する
  • 午後とのつながり:最高。セキュアプログラミング問題に直結
メールセキュリティ A

毎回1.5問前後。用語の丸暗記だけで確実に取れる得点源。

  • 典型問題:SPF(3回)、DKIM(3回)、DMARC、SMTP-AUTH(5回)、OP25B(3回)、サブミッションポート587、メール暗号化3方式の鍵の単位
  • ひっかけ:SPF(IP)とDKIM(署名)の検証対象の混同。S/MIMEとSMTP over TLSの混同
  • 対策:「SPF=IP、DKIM=署名、DMARC=ポリシー」を呪文化
  • 午後とのつながり:高。送信ドメイン認証の設定問題が午後に出る
DoS/DDoS・ボットネット B

毎回1問前後でやや増加。手口の違いだけ問われる。

  • 典型問題:Smurf(6回)、ICMP Flood(4回)、NTPリフレクション、DRDoS、バックスキャッタ、Mirai、NOTICE
  • ひっかけ:直接型と反射型の区別。Smurfは「応答を攻撃対象に集中させる」
  • 対策:DoS系比較メモ1枚で全部取れる
  • 午後とのつながり:中
マルウェア・ランサムウェア・標的型攻撃 B

毎回2問前後。R5以降の新顔用語の再出題ゾーン。

  • 典型問題:Mirai(3回)、ルートキット、ポリモーフィック型、ビヘイビア法、証拠保全の優先順位、クリプトジャッキング、コネクトバック、AI攻撃
  • ひっかけ:マルウェアの型の定義混同。証拠保全は揮発性の高い順
  • 対策:新顔(ドメインフロンティング/モデルインバージョン等)は一言定義を暗記
  • 午後とのつながり:高。インシデント対応・ログ調査の長文に直結
セキュア開発・脆弱性管理・CVE/CVSS B

毎回0.5〜1問で増加傾向。混同整理だけで取れる。

  • 典型問題:CVE(4回)、CWE、CVSS、JVN、SBOM、ペネトレーションテスト、ファジング
  • ひっかけ:CVE(識別子)/CWE(種類)/CVSS(深刻度)の相互混同
  • 対策:3つの違いを一言で言えるようにする
  • 午後とのつながり:中。脆弱性管理・パッチ運用の問題
ISMS・リスク管理・監査・法制度 B

毎回2.5問前後で増加傾向。問25の監査枠は毎回固定。

  • 典型問題:JIS Q 27000用語、ISMAP/ISMAP-LIU、NIST CSF 2.0(GOVERN追加)、MITRE、SIM3、CSIRT/PSIRT/SOC、内部統制、システム監査基準
  • ひっかけ:全般統制(開発保守の管理)と業務処理統制(アプリ内の統制)の区別
  • 対策:制度名と「どこが・何のために」をセットで暗記
  • 午後とのつながり:中。リスクアセスメントの考え方は午後でも使う
クラウド・ゼロトラスト・ログ管理・SIEM B

ほぼ全てR4以降の出題。最も明確な増加分野で、初出→2回目の再出題が狙われる時期。

  • 典型問題:CASB(利用可視化)、CSPM(設定監視)、SOAR(運用自動化)、UEBA(振る舞い検知)、IoC(侵害の痕跡)
  • ひっかけ:5つの機能の相互混同
  • 対策:一言定義の暗記のみで取れる
  • 午後とのつながり:中〜高。クラウド設定不備は午後の新定番
ネットワーク(FW・IDS/IPS含む) B

毎回4〜5問(問18〜20の固定枠+セキュリティ機器)。

  • 典型問題:ステートフルインスペクション(計8回)、IPS(シグネチャ/アノマリ)、サブネット計算、スパニングツリー、ICMP、DNSSEC(計8回)、カミンスキー攻撃対策、無線LAN
  • ひっかけ:TCPヘッダーとIPヘッダーの情報の混同(宛先ポート番号はTCP)
  • 対策:計算問題は手を動かす。FW/IDS/IPS/WAFは比較で暗記
  • 午後とのつながり:高。FWルール設計・ログ調査の基礎
データベース(問21) / 開発技術・SM(問22〜24) C

応用情報レベルの固定枠。ここで5問以上落とさないことが合格の前提。

  • 典型問題:SQL実行結果、コミット完了時点(ログ書込み時)、テスト駆動開発、KPT、可用性計算(R7春98.3%・R7秋99.52%)、JIS Q 20000
  • ひっかけ:可用性計算でサービス提供時間帯の条件を見落とす
  • 対策:可用性計算だけ2〜3問練習。他は応用情報の記憶で対応
  • 午後とのつながり:低

6割戦略・7割戦略

捨てない分野

  • 暗号・PKI・TLS(毎回4〜5問。再出題の中心)
  • メールセキュリティ(丸暗記で確実)
  • 問18〜25の8問(応用情報レベル)

暗記で取る分野

  • DoS系の手口の違い
  • マルウェア定番(Mirai/ルートキット/ビヘイビア法)
  • クラウド・SOC系用語(CASB/CSPM/SOAR/UEBA/IoC)
  • 制度系(ISMAP/NIST CSF/CVE・CWE・CVSS)

深追いしない分野

  • DBの細かい理論(問21の1問だけ)
  • 開発技術の細部(問22〜23)
  • 初見の長文計算問題(最後に回す)

得点内訳の目安:再出題8〜12問+新傾向用語2〜3問+問18〜25で5〜6問 → 15問ラインは過去問演習の量でほぼ決まる。

仕組み理解が必要

  • SAML/OAuth2.0の流れ(登場人物を図で)
  • TLS・証明書検証の流れ
  • XSS/CSRF/SQLi(攻撃→だから対策)
  • サブネット・可用性計算(手を動かす)

午後にも効く分野

  • Web脆弱性、メール認証(SPF/DKIM/DMARC)
  • TLS/証明書、認証連携
  • FW/IDS/IPSのログ読解
  • インシデント対応(証拠保全の順序)

重点復習

  • R6〜R7初出(次回最有力):ドメインフロンティング、コネクトバック、IoC、UEBA、CSPM、SIM3、Identity Proofing、DTLS、Enhanced Open、モデルインバージョン、パスキー
  • サイクル的に有力:衝突発見困難性、OCSP、DNSSEC、SMTP-AUTH

混同しやすい用語の違い

ひっかけ選択肢はほぼここから作られる。1枚30秒で確認。詳細:sc_am2_terms_diff.md

SPF / DKIM / DMARC

SPF:送信元IPを検証(DNSのTXTと照合。エンベロープFrom)
DKIM:デジタル署名を検証(転送に強い)
DMARC:失敗時の処理方針を送信側が指定+レポート
覚え方:「SPFはIP、DKIMは署名、DMARCはポリシー」

IDS / IPS / WAF

IDS:検知して通知(遮断しない)
IPS:インラインで検知+遮断
WAF:Webアプリ層(HTTP)に特化して防御
シグネチャ型=既知パターン照合 / アノマリ型=正常からの逸脱検知

ハッシュ / MAC / デジタル署名

ハッシュ:鍵なし。完全性のみ
MAC:共通鍵。完全性+相手確認(否認防止は不可)
署名:秘密鍵で署名。完全性+否認防止
覚え方:「鍵なし→共通鍵→公開鍵」の順に保証が強くなる

共通鍵暗号 / 公開鍵暗号

共通鍵:同じ鍵。高速。鍵配送が課題。n人でn(n-1)/2個
公開鍵:鍵ペア。暗号化=相手の公開鍵、署名=自分の秘密鍵。低速
実運用はハイブリッド:データは共通鍵、共通鍵の受け渡しに公開鍵

CRL / OCSP

CRL:失効証明書のシリアル番号一覧をCAが定期発行
OCSP:失効状態をリアルタイム照会(応答=VA)
載るのは「有効期限内に失効」した証明書。期限切れは対象外

SAML / OAuth

SAML:認証の連携(SSO)。IdP→アサーション→SP
OAuth 2.0:認可の委譲。アクセストークンで限定アクセス
覚え方:「SAML=あなたは誰か、OAuth=何をしてよいか」

XSS / CSRF / SQLインジェクション

XSS:被害者のブラウザでスクリプト実行 → エスケープ・CSP
CSRF:意図しないリクエストを送信させる → トークン・再認証
SQLi:SQL文を改変 → プレースホルダ
CSRF対策に「エスケープ」を選ばせるのが定番ひっかけ(3回出題)

TCP / UDP

TCP:コネクション型(3ウェイハンドシェイク)。信頼性あり
UDP:コネクションレス。高速。送信元偽装が容易→反射攻撃に悪用
ポート番号はTCP/UDPヘッダー、IPアドレスはIPヘッダー(3回出題)

DoS / DDoS(と反射型)

DoS:単一発信元からの妨害
DDoS:多数の踏み台(ボットネット)から分散攻撃
DRDoS:送信元を標的に偽装→応答が標的に集中(DNS/NTP増幅)
Smurf=ICMPエコー要求をブロードキャストへ(送信元は標的に偽装)

CVE / CVSS(+CWE)

CVE:個々の脆弱性の識別子(MITREが事務局)
CVSS:深刻度の数値評価(0.0〜10.0)
CWE:脆弱性の種類の一覧(XSS=CWE-79)
覚え方:「CVE=名前、CVSS=点数、CWE=種類」

認証 / 認可

認証:相手が本人かを確認(パスワード・生体・証明書)
認可:認証済みの主体に何を許すか(アクセス制御)
+Accounting(利用記録)でAAA(RADIUS/Diameter。R6秋出題)

暗号化 / ハッシュ化 / 符号化

暗号化:鍵があれば元に戻せる(機密性)
ハッシュ化:元に戻せない一方向変換(完全性)
符号化:表現形式の変換。誰でも戻せる(Base64等)
「Base64で暗号化」という表現は常に誤り
おまけ:直近試験で問われた新しめの対比
  • CASB(利用の可視化) / CSPM(設定の監視) / SIEM(ログ相関分析) / SOAR(対応自動化) / UEBA(振る舞い検知)
  • CSIRT(自組織のインシデント対応) / PSIRT(自社製品の脆弱性対応。R7秋出題) / SOC(監視・分析の実働部隊)
  • 量子暗号(QKD)(盗聴を検知できる鍵配送) / PQC(量子計算機に耐える従来型暗号)
  • サイバーキルチェーン(攻撃の段階モデル) / MITRE ATT&CK(戦術・技術のナレッジベース)
