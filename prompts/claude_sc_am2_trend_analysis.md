# Claude Code 投入用プロンプト：SC午前Ⅱ 傾向分析・対策資料作成

## 目的
このリポジトリ内の情報処理安全確保支援士試験（SC）の過去問データを使って、**午前Ⅱ試験だけ**の出題傾向を分析し、合格点を取るための対策資料を作成してください。

ユーザーは応用情報技術者試験に合格済みです。したがって、応用情報合格などで免除される**午前Ⅰ／高度共通午前Ⅰ**は分析対象に含めないでください。

---

## 最重要ルール

### 分析対象に含めるもの
- 情報処理安全確保支援士試験 午前Ⅱ
- ファイル名に以下を含むものを優先して調査する
  - `sc_am2`
  - `午前Ⅱ`
  - `午前II`
  - `情報処理安全確保支援士試験 午前Ⅱ`

### 分析対象から除外するもの
以下は必ず除外してください。

- 午前Ⅰ
- 午前I
- 高度共通午前Ⅰ
- 高度共通午前I
- `koudo_am1`
- `am1`
- 応用情報合格で免除される範囲
- 午後Ⅰ、午後Ⅱ、科目B、午後問題

ただし、午後問題との関連性を説明する目的で「午後にもつながる分野」として言及するのは可です。集計対象には含めないでください。

---

## 前提
SC午前Ⅱは、暗号、PKI、TLS、Web脆弱性、メールセキュリティ、ネットワーク、マルウェア、リスク管理など、SC固有の知識が中心です。

ユーザーは1日30分程度で学習する想定です。よって、単なる網羅ではなく、**出題頻度・再出題可能性・午後との接続性**を基準に優先順位を付けてください。

---

## 作業手順

### 1. リポジトリ構成の確認
まず、リポジトリ内のファイル構成を確認してください。

確認対象例：

```text
**/*sc_am2*
**/*午前Ⅱ*
**/*午前II*
**/*SC*AM2*
**/*SC*午前*
```

その上で、分析対象ファイル一覧を作成してください。

出力例：

```md
## 分析対象ファイル
- 2017h29h_sc_am2_qs.pdf
- 2017h29h_sc_am2_ans.pdf
- 2017h29a_sc_am2_qs.pdf
- 2017h29a_sc_am2_ans.pdf
...
```

除外したファイルも別に記録してください。

```md
## 除外ファイル
- 2017h29h_koudo_am1_qs.pdf：午前Ⅰのため除外
- 2017h29h_koudo_am1_ans.pdf：午前Ⅰのため除外
- 2017h29h_sc_pm1_qs.pdf：午後Ⅰのため除外
```

---

### 2. 午前Ⅱ問題の抽出
各回の午前Ⅱ問題について、可能な範囲で以下を抽出してください。

- 年度
- 春期／秋期
- 問番号
- 正解
- 問われた用語
- 問題の主題
- 大分類
- 小分類
- ひっかけポイント
- 暗記で解けるか、仕組み理解が必要か
- 午後試験との関連度
- 再出題可能性

PDFの本文抽出が難しい場合は、ファイル名・解答例・既存Markdown・OCR済みテキスト・過去の変換済みファイルを優先して使ってください。

最低限、以下のカテゴリで分類してください。必要に応じて追加して構いません。

```text

認証・認可・SAML・OAuth

ネットワーク：TCP/IP、DNS、ARP、無線LAN、FW、IDS/IPS

DoS/DDoS・ボットネット
セキュア開発・脆弱性管理・CVE・CVSS
ISMS・リスク管理・監査・法制度
クラウド・ゼロトラスト・ログ管理・SIEM
その他SC午前Ⅱ固有テーマ
```

---

### 4. 優先順位付け
次の観点で、各カテゴリに優先度を付けてください。

```text
A：最優先。頻出で午後にもつながる。短期で得点源にする。
B：重要。午前Ⅱで出やすい。暗記と理解を並行する。
C：余力。出題はあるが優先度はやや下がる。
D：低頻度。直前確認でよい。
```

優先度判定では、単純な出題数だけでなく、次も考慮してください。

- 直近回での出題有無
- 類似問題の再出題可能性
- 午後問題との接続性
- 忘れやすさ
- 用語の混同しやすさ
- 1問取るまでの学習コスト

## 分析時の注意

### 1. 午前Ⅰを混ぜない
この作業で最も重要です。

ファイル名に `koudo_am1` が含まれるものは、SC受験者に共通する午前Ⅰの資料です。今回の分析対象ではありません。

### 2. 単なる頻度集計で終わらせない
以下まで書いてください。

- なぜ出るのか
- どう問われるのか
- 何と混同しやすいのか
- どの順番で覚えるべきか
- 午後にも効くのか

### 3. 1日30分で実行できる粒度にする
「暗号を勉強する」では広すぎます。

悪い例：

```md
暗号分野を勉強する
```

良い例：

```md
1日目：共通鍵暗号・公開鍵暗号・ハイブリッド暗号の違いを整理し、午前Ⅱを5問解く
2日目：ハッシュ・MAC・デジタル署名の違いをカード化し、間違えた選択肢を記録する
```

### 4. スマホで読めるMarkdownにする
横長の表は最小限にしてください。

- 長い表はCSVに出す
- Markdownは要点中心
- 見出しを細かく分ける
- 箇条書きを短くする
