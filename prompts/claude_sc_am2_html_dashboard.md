# Claude Code 投入用プロンプト：SC午前Ⅱ HTMLダッシュボード化

## 目的
既に作成済みのSC午前Ⅱ分析資料を、スマホ・PCの両方で読みやすい**単一HTMLページ**に変換してください。

対象は、情報処理安全確保支援士試験（SC）の**午前Ⅱ**だけです。午前Ⅰ・高度共通午前Ⅰ・`koudo_am1`・午後問題は集計対象に含めないでください。

---

## 既存成果物
以下のファイルを読み込んで、HTML化してください。

```text
docs/sc_am2_trend.md
docs/sc_am2_strategy.md
docs/sc_am2_flashcards.md
docs/sc_am2_30min_plan.md
docs/sc_am2_terms_diff.md
data/sc_am2_topic_summary.csv
data/sc_am2_questions.csv
```

---

## 作成するファイル

### 必須

```text
docs/sc_am2_index.html
```

### 任意。ただし保守性が上がるなら作成可

```text
docs/assets/sc_am2_style.css
docs/assets/sc_am2_app.js
```

GitHub Pagesでそのまま見られるようにしてください。外部CDNに依存しないでください。ネット接続なしでも表示できる構成にしてください。

## HTMLの構成

`docs/sc_am2_index.html` は、次の構成にしてください。

```text
1. ヘッダー
2. 結論サマリー
3. 今日やること
4. 優先順位マップ
5. カテゴリ別傾向
6. 6割戦略・7割戦略
7. 暗記カード
8. 用語の違い
9. 4週間学習計画
10. 問題データ一覧
11. フッター
```

---

## 1. ヘッダー

表示内容：

- タイトル：`SC午前Ⅱ 傾向と対策`
- サブタイトル：`応用情報合格者向け・午前Ⅰ免除前提`
- 対象範囲：`SC午前Ⅱのみ / 午前Ⅰ・午後は集計対象外`
- 最終更新日：HTML生成時の日付

---

## 2. 結論サマリー

一番上に、3〜5枚のカードで表示してください。

例：

```text
最優先：Web脆弱性、メールセキュリティ、PKI/TLS
目標：25問中17〜19問
最低ライン：15問
方針：暗記カード → 過去問 → 間違い復習
```

---

## 3. 今日やること

1日30分で何をすればいいかを、ファーストビュー付近に表示してください。

表示例：

```text
今日の30分
1. 5分：昨日間違えた用語を確認
2. 15分：午前Ⅱを10問解く
3. 10分：間違えた理由を1行で記録
```

---

## 4. 優先順位マップ

`data/sc_am2_topic_summary.csv` を読み取り、カテゴリ別にカード表示してください。

カードに表示する項目：

```text
カテゴリ名
出題数
直近重みスコア
優先度 A/B/C/D
午後関連 high/middle/low
学習アクション
```

見せ方：

- 優先度Aは目立つカード
- 優先度Bは通常カード
- C/Dは折りたたみ、または下部表示
- スマホでは1列
- PCでは2〜3列

## 5. カテゴリ別傾向

`docs/sc_am2_trend.md` の内容を元に、カテゴリごとに整理してください。

各カテゴリは、以下の形にしてください。

```html
<section class="topic-section">
  <h2>Web脆弱性</h2>
  <p>要約</p>
  <ul>
    <li>典型問題</li>
    <li>ひっかけ</li>
    <li>対策</li>
    <li>午後とのつながり</li>
  </ul>
</section>
```

横長表をそのまま貼らないでください。スマホで読めるように、カード・箇条書き中心にしてください。

## デザイン要件

### 全体

- スマホ最優先
- PCでも横幅を活かす
- 文字サイズは小さくしすぎない
- 余白を十分に取る
- 長文は折りたたむ
- 表はカード化する

### 色

派手にしすぎないでください。学習用として読みやすい配色にしてください。

推奨：

```text
背景：#f6f8fb
本文：#1f2937
カード背景：#ffffff
アクセント：#2563eb
注意：#dc2626
優先度A：#fee2e2
優先度B：#dbeafe
優先度C：#ecfdf5
優先度D：#f3f4f6
```

### レスポンシブ

必須ブレークポイント：

```css
@media (max-width: 640px) { ... }
@media (min-width: 641px) and (max-width: 1024px) { ... }
@media (min-width: 1025px) { ... }
```

## 機能要件

JavaScriptを使ってよいですが、外部ライブラリは禁止です。

最低限ほしい機能：

```text
- ページ内ナビゲーション
- トップへ戻るボタン
- 暗記カード検索
- 問題一覧検索
- 優先度フィルタ
- カテゴリフィルタ
- 折りたたみ
```

可能なら追加：

```text
- localStorageで「覚えた」チェックを保存
- localStorageで最後に開いたセクションを保存
- 印刷用CSS
```

## GitHub Pages対応

GitHub Pagesで開けるように、相対パスを使ってください。

例：

```html
<a href="./sc_am2_trend.md">傾向分析Markdown</a>
<a href="../data/sc_am2_questions.csv">問題別CSV</a>
```

`docs/sc_am2_index.html` を直接ブラウザで開いても崩れないようにしてください。

---

## 既存Markdownの扱い

MarkdownをそのままHTML化するだけでは不十分です。

悪い例：

```text
MarkdownをHTMLに変換しただけ
巨大な表をそのまま表示
スマホで横スクロールが多い
```

良い例：

```text
結論を先頭に出す
カードで要点を表示
詳細は折りたたむ
検索・絞り込みができる
CSVの大きなデータは必要な範囲だけ見せる
```
